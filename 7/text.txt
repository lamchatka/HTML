07. HTML изображения — тег img. Относительные и абсолютные пути

Для добавления изображения на страницу есть несколько способом. 
Один из самых явных -  использования HTML тега img.

Тег является одиночным и принимает один обязательный атрибут - src. 
Значением атрибута является путь до файла изображения любого формата, который поддерживает браузер.
Помимо атрибута src хорошим тоном является задавать атрибут alt - описание изображения.
Этот атрибут при обычных условиях виден пользователям не будет, 
но если у пользователя отключится интернет
или путь к файлу не корректен пользователь увидит значение alt.
Также люди с ограниченными возможностями, использующие screen readers смогут понять,
что изображено на картинке.
Если изображение - декоративный элемент, то alt="", но alt должен быть указан всегда.

Также хорошей практикой является указание ширины и высоты изображения через атрибуты width и height.

Зачем указывать значения высоты и ширины в разметке, если для этого есть CSS?

Коротко: медифайлы загружаются дольше, через HTML страница, поэтому тег img на странице уже существует размером 0px,
и блок, идущий ниже изображения как бы смещается наверх.
в ином случае, если img указаны атрибуты widht и height место под элемент зарезервированно и нет непредвиденных скачков в интрефейсе.

Атрибут loading: eger, lazy.
При загрузке HTML страницы браузером по умолчанию сразу загружается все изображения из всех тегов img на странице. 
Если в вне области видимости пользователя есть изображения, находящиеся за пределами экрана (видно только при прокрутке), 
то неоптимально тратить ресурси сети на загрузку всех изображений сразу. Для этих целей есть механизм lazy loading или ленивая нагрузка.
Позволяет загружать те медиаресурсы, в нашем случае изображения, которые должны оказать перед глазами пользователя при загрузке сайта.
Изображения, находящиеся вне экрана на других экранов, за скролом, можно прогрузить попозже.
Раньше механизм реализовывался силами JS, но теперь все проще.
Значения eger, lazy, по умолчанию eger. Как только пользователь долистает до изображение на другом экране, то оно будет загржено.

Относительные и абсолютные пути.

Путь - адрес до какого-либо ресурса.

Относительный: от текущего файла до требуемого - ./ или ../ 
Абсолютный путь: Адрес ресурса относительно корня сайта - корень сайта это главная папка проекта.
