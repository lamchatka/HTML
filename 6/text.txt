06. HTML ссылки и кнопки — теги a, button. Якорная ссылка, ссылка mailto и tel. Атрибут target _blank

Тег a.
Для навигации по страницам сайта и за его пределами используется тег a.
Тег a позволяет сделать текст, изображение или любой другой элемент ссылками.
Достаточно лишь обернуть фрагмент текста тегом a с обязательным атрибутом href.
Этот атрибут должен содержать адрес, при клике на который произойдет переход на другую страницу
или элемент. Ссылка может вести на страницу на сайте, в интернете, на любой элемент на странице,
на номер, адрес почты и тд.

Разберем эти нестандартные примеры.
1. Якорная ссылка - ссылка, которая указывает на элемент конкретной HTML страницы (в рамках одной страницы).
Для использования якорной ссылки в атрибуте href через специальный символ # должен быть указан
идентификатор (id) элемента, на который мы хотим попасть при клике на ссылку.

2. Ссылка на файл - в атрибуте href указать ссылку на конкретный файл и 
добавить одиночный атрибут download. Начнется загрузка файла.

3. Ссылка на email - адрес email оборачивается тегом a, 
в атрибуте href добавляется специальный префикс "mailto:" + адрес email для перехода в почтовый клиент.

4. Ссылка на телефон - номер телефона оборачивается тегом a,
в атрибуте href добавляется специальный префикс "tel:" + номер, 
чтобы у пользователя перекинуло на звонок по сотовой связи. 
Не рекомендуется в атрибуте использовать любые символы, кроме +.

5. Открытие ссылки в новой вкладке при клике.
Указать в теге a допольнительный атрибут target со значением _blank.

Кнопка, тег button.
Единственное назначение - обработка клика и последующее выполнение логики.
У кнопки есть важный атрибут - type, принимающий три значения: button, submit, reset.
Атрибут type=button стоит прописывать всем кнопкам по умолчанию, 
так как без его указания кнопка считывается с атрибутом submit. 

Атрибут type=submit используется, если кнопка лежит внутри формы 
и подразумевает отправку на сервер каких-либо введенных пользователем данных на сервер.
Пример: кнопка войти в форме авторизации.

Атрибут type=reset используется внутри форм и
сбрасывает какие-либо значения, введеные пользователем, до значений по умолчанию.
Пример: кнопка сбросить фильтр в форме фильтров каталогов товаров.

Почему стоит указывать button type button?
 <form>
        <input type="password" />

        <button>"иконка с глазом"</button>

        <button type="sumbit">Войти</button>
    </form>

Если в форме есть кнопки, которые не reset и не sumbit и если не указать для них type=button, 
при они будут восприняты браузером как type=sumbit. 
Произошла перезагрузка страницы и форма отправлена.

<form>
        <input type="password" />

        <button type="button">"иконка с глазом"</button>

        <button type="sumbit">Войти</button>
    </form>


Дилемма: ссылка или кнопка.
Создавая фронтенд приложение при работе с макетом часто непонятно, 
какой элемент будет ссылкой, какой кнопкой.
Советую придерживаться простого правила:
Если при нажатии на элемент должен просходит переход на другую страницу, то используй тег a.
Если при нажатии прозойдет действие в рамках текущей страницы, то используй тег button.
