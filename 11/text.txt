11. HTML формы. Теги form, fieldset, legend, label и поле ввода input

Формы и поля ввода - это главные интерактивные поля любого сайта. 
Пользователи постоянно вводят какие-то данные, жмут на чекбоксы,
соглашаясь с условиями, тыкают на радиокнопки, выбирая один из вариантов ответа, 
опции из выпадающих списков и т.д.

Под каждый вышеприведенный сценарий и не только в HTML есть соответствующий тег и
набор атрибутов. В следующих видео мы разберем большинство из них.

Тег form.
Этим тегом оборачивается совокупность тегов, 
с которыми взаимодействует пользователь.
У этого тега есть два важных уникальных атрибута - action и method.
В значение action указывают адрес скрипта обработчика, 
расположенного на стороне сервера, который обработает форму при ее отправке.
В method указывается метод отправки формы. По умолчанию, если не указать этот атрибут,
то у формы будет метод POST.

Тег fieldset.
Служит для визуальной группировки нескольких полей ввода в один набор.
По умолчанию браузер добавляет границу для этого тега, которую можно убрать через CSS.

<form>
        <fieldset>
            Группа полей ввода 1
        </fieldset>
        <fieldset>
            Группа полей ввода 2
        </fieldset>
        <fieldset>
            Группа полей ввода 3
        </fieldset>
</form>

Тег legend.
Используется в качестве заголовка внутри тега fieldset.
Браузер по умолчанию располагает заголовок слева сверху на уровне рамки fieldset.

Тег label.
Это тег метки для интерактивного поля ввода. Используется для семантики в комплексе с тегом input.
Обязательно использовать, если есть input!!!
    
    <label>Логин:</label>

Тег input.
Самый сложный и многогранный для пользовательского взаимодействия.
Является одиночным тегом.
Если посмотреть результат отображения тега input в браузере, 
то отобразиться пустой прямоугольник.
Если нажать на пустой прямоугольник, то внутри прямоугольника появится курсор
и выделится граница этого прямоугольника.

 <input />

Тег label в связке с input. Атрибуты for и id.
Для правильно связи метки и поля ввода в теге label указать атрибут for
со значением индентификатора поля ввода. То есть, у input есть атрибут id,
и его значение должно быть использовано в атрибуте for тега label.

<label for="login">Логин:</label>
<input id="login"/>

Особенности: 
1. id должен быть уникальным в рамках страницы.
2. При нажатии на текст заключенный в label происходит 
вход в режим ввода значения в input.
3. Screen reader прочитывает label

Атрибут name.
Очень важен для полей формы (скорее всего ввода). Его значение определяет имя элемента формы, 
которое необходимо серверу для распознавания введенных пользователем данных.
При отсутствии этого атрибута сервер не увидит данные, считанные с этого поля.

<label for="login">Логин:</label>
<input id="login" value="qwerty"/>


Атрибут value.
Служит для нескольких целей:
1. В тех input, в которые юзер может ввести значения, 
значение атрибута value в разметке будет отображено в поле ввода по умолчанию.
2. Другие виды input"ов будут рассмотрены позже.

<label for="login">Логин:</label>
<input id="login" value="qwerty" name="user-login"/>

Атрибут type.
Настройка вариации атрибутов происходит благодаря атрибуту type.
Значение по умолчанию - если не указывать этот атрибут - type=text. Он сигнализирует,
что input будет обычным текстовым полем.
У атрибута type есть ряд альтернативных значений, которые задают input
особое назчание, функционал и внешний вид.
